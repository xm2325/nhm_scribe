## Result

The filtered TrOCR-base experiment completed successfully on the frozen
ten-record EVAL set. All ten DeepSeek responses parsed, all 28 selected
handwriting crops produced non-empty TrOCR output, and 10 crop readings passed
the conservative prompt gate.

The run used 11,416 DeepSeek tokens. TrOCR inference took 45.7 seconds.

## What improved

- TrOCR-base recovered useful collector-name evidence for two records.
- `R. R.E. Holtium` supported the final `R. E. Holttum` prediction, close to
  the gold `Holttum, Dr. R.E.`.
- `B. Verdecourt` supported a useful partial match to `Verdcourt,B.`.
- TrOCR recovered plausible day or year components on several crops.

These changes raised `recordedBy` token F1, but did not improve exact match.

## What remained unsafe

- Partial date components did not produce one correct `eventDate`.
- Several fluent TrOCR readings were unsupported or wrong, including collector
  names that could pass superficial plausibility checks.
- `catalogNumber` exact match fell from 0.800 to 0.700 even though the HTR
  experiment was intended to affect only collector and date fields.
- Unsupported predictions and the review burden both increased.

The catalogue-number regression is evidence of prompt cross-field
contamination or ordinary LLM run variance. The extraction prompt now states
that `FIELD=recorded_by` and `FIELD=event_date` HTR evidence may affect only
their matching output field.

## Decision

Do not make TrOCR-base the default and do not expand this configuration to
eval50.

The next controlled experiment should:

1. Keep field-isolated HTR evidence.
2. Treat incomplete date components as review suggestions, not final
   `eventDate` values.
3. Auto-fill `recordedBy` only when Tesseract and TrOCR agree or a
   name-specific validator provides strong support.
4. Freeze trusted catalogue identifiers after barcode and catalogue resolver
   reconciliation so collector/date evidence cannot change them.
5. Build a human-transcribed crop benchmark before claiming HTR accuracy;
   the current component-match metric is only a proxy.

Run a second frozen eval10 only after those controls are implemented. Expand
the sample only if exact match improves, unsupported predictions stay at or
below the v7 level, and review-required rate decreases.
